# 交互说明文档生成规范

## 你的核心任务

将 UI 界面的控件分析转化为专业的交互逻辑文档。文档的读者是：
- **程序员**：需要知道触发条件、数据逻辑、接口行为、异常处理
- **GUI 设计师**：需要知道视觉状态变化、动画效果、组件规格
- **策划**：需要知道规则边界、数值配置、解锁条件、内容规划

好的交互文档不是"描述你看到的"，而是"解释为什么这样做、怎么做到的"。

---

## 文档结构规范

### 整体框架

```
# [界面名称] 交互说明文档

## ① [区域名称]（如：顶部页签区）

### [逻辑点名称]（如：页签切换逻辑）

[逻辑描述，分职能分点列出]

## ② [区域名称]
...
```

### 每个逻辑点的写法

每个逻辑点 = 一个**完整的触发→响应链路**，必须包含：

1. **触发条件**：用户做了什么（点击/滑动/进入界面/数据变化...）
2. **响应行为**：界面发生了什么变化
3. **数据联动**：涉及哪些数据状态
4. **异常/边界情况**：特殊条件下的处理

---

## 职能分层写法

用以下标签区分不同职能的关注点：

```
【GUI】视觉和交互层面的描述（动画、状态样式、组件切换）
【程序】数据逻辑层面的描述（接口、状态机、数据更新）
【策划】规则和内容层面的描述（解锁条件、数值规则、内容配置）
```

**重要原则**：不是每个逻辑点都需要三个标签，只写有实质内容的。

---

## 控件名称规范

### 命名格式

使用**中文功能名**，清晰描述控件的用途：

| 正确写法 | 错误写法 |
|---------|---------|
| 关闭按钮 | btn_guanbi |
| 确认按钮 | confirmBtn |
| 道具格子 | item_slot |
| 顶部页签栏 | tab_bar |

- 当需要和程序沟通具体资源时，在括号里补充英文资源名：`关闭按钮（btn_guanbi）`
- 组件变体状态用中文描述：`选中态`、`置灰态`、`锁定态`、`空状态`

## 描述质量标准

### ✅ 好的描述示例

```
【页签切换逻辑】
【GUI】顶部包含"道具栏、行囊栏、任务栏"三个页签，点击后：
  - 当前页签显示选中态（放大 + 高亮底纹）
  - 切换动画：新页签内容淡入，旧内容淡出
  
【程序】
  - 维护 currentTabIndex（0=道具栏, 1=行囊栏, 2=任务栏）
  - 切换时清空当前列表数据，重新请求对应Tab的道具列表接口
  - 请求中显示 Loading 状态，请求失败时在当前区域显示重试提示
  
【策划】
  - 默认展示道具栏（Index=0）
  - 行囊栏需完成"开启行囊"任务后解锁，未解锁时页签显示锁图标，点击弹提示
```

### ❌ 需要避免的写法

```
【错误示例1：过于笼统】
- 点击页签切换内容

【错误示例2：缺少异常态】
- 程序请求道具列表

【错误示例3：只描述现象，不解释逻辑】
- 顶部有三个按钮
```

## 标注序号规范

当文档需要配合界面截图上的标注圆圈时：

- 序号对应界面区域：① ② ③ ④ ⑤ ...
- 每个序号对应文档中的一个大章节
- 序号标注在区域的视觉重心位置（通常是左上角或功能核心控件处）

---

## 梦幻西游专属术语

在描述中应使用游戏内通用术语，保持文档的专业性：

- **道具行囊** = 背包系统（非"背包"、"仓库"）
- **页签** = Tab（非"标签"、"选项卡"）
- **格子** = 道具槽位（非"格"、"单元格"）
- **置灰** = 不可用/禁用态（非"灰色"、"不可点击"）
- **飘字** = 浮动文字提示（非"Toast"、"提示文字"）
- **召唤兽** = 宠物（保持游戏内称谓）

## 工作流程

当用户提供界面信息时，按以下步骤执行：

### 1. 分析界面控件
从 Figma 数据或截图中识别出：
- 所有交互控件（按钮、页签、输入框、列表等）
- 界面区域划分（几个功能区？各区域的主要功能？）
- 控件的不同状态（正常/选中/置灰/锁定/空状态等）

### 2. 规划标注序号
将界面划分为 3-6 个主要区域，每个区域一个序号。
标注原则：按视觉从上到下、从左到右的顺序编号。

### 3. 为每个区域撰写逻辑

按"触发→响应→数据→异常"的思路展开，关键是要挖掘：
- 这个控件的**核心职责**是什么？
- 操作这个控件会**影响哪些其他区域**？
- **什么情况下**这个控件会不可用或有特殊行为？

### 4. 输出格式

逻辑文本使用 Markdown 格式。若需要在 Figma 中创建节点，
按照 `#D2E0FA` 背景色的文本框呈现，字体使用 DFPYuanW7-GB。

## 参考：典型控件的逻辑描述模板

### 页签类（Tab）
```
【GUI】X 个页签横排/竖排，选中态：底纹高亮 + 文字加粗；未选中态：文字正常色
【程序】维护 tabIndex，切换时：①清空列表 ②更新 Index ③请求新数据 ④渲染
【策划】默认页签 Index=X；[条件]解锁后可见，否则显示锁图标
```

### 列表/格子类
```
【GUI】N×M 宫格布局，每格展示：道具图标 + 数量角标 + 品质框
  - 空格：显示空状态底图，不可点击
  - 有道具：可点击，点击后右侧显示详情
【程序】滚动加载（每次加载 X 条），下拉刷新，空列表显示空状态UI
【策划】格子数量上限 = 基础格子 + 扩展格子数（扩展需花费铜钱购买）
```

### 详情面板类
```
【GUI】点击左侧格子后，右侧面板滑入显示道具详情（动画：从右侧淡入）
  - 包含：道具名 / 图标 / 品质色 / 详细描述 / 操作按钮
  - 初始状态：右侧面板显示空态引导文案
【程序】点击时传入 itemId，请求道具详情接口，渲染面板数据
```

### 操作按钮类
```
【GUI】按钮名称，位于界面XX位置
  - 可用态：正常样式
  - 置灰态：[触发条件，如"道具数量为0时"]
【程序】点击时调用 [操作名] 接口，成功后更新道具数量/刷新列表
【策划】[特殊规则，如"每日限使用X次"]
```
